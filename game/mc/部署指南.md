### 部署到 GitHub Pages

#### 步骤 1：创建 GitHub 仓库
1. 访问 https://github.com/new
2. 创建新仓库（例如：minecraft-shop）
3. 初始化为空仓库

#### 步骤 2：上传文件
```bash
cd /Users/RubinCai_1/WorkBuddy/mc
git init
git add .
git commit -m "Initial commit - Minecraft Shop Game"
git branch -M main
git remote add origin https://github.com/你的用户名/minecraft-shop.git
git push -u origin main
```

#### 步骤 3：启用 GitHub Pages
1. 进入仓库的 Settings（设置）
2. 左侧菜单找到 "Pages"
3. Source 选择 "Deploy from a branch"
4. Branch 选择 "main" 文件夹选择 "/ (root)"
5. 点击 Save

#### 步骤 4：等待部署
- 几分钟后，GitHub 会给你一个访问地址
- 例如：`https://你的用户名.github.io/minecraft-shop/`

### 🌐 其他部署平台

如果 GitHub Pages 有问题，可以使用：

1. **Vercel**（推荐）
   - 访问 https://vercel.com
   - 连接 GitHub 仓库
   - 自动部署，速度快

2. **Netlify**
   - 访问 https://netlify.com
   - 拖拽 mc 文件夹到网站
   - 即刻上线

3. **Cloudflare Pages**
   - 访问 https://pages.cloudflare.com
   - 连接 GitHub 仓库
   - 全球 CDN 加速

### ✅ 为什么上传后可以正常工作？

1. **静态文件服务**：这些平台都提供标准的 HTTP 静态文件服务
2. **正确的 MIME 类型**：会正确识别 .js 文件为 JavaScript
3. **跨域处理**：现代 CDN 都正确处理 CORS 头
4. **HTTPS 支持**：提供安全的 HTTPS 访问

### 📝 注意事项

1. 确保 mc 文件夹下包含所有必需文件：
   - index.html
   - game.js
   - data.js
   - assets/ 文件夹（如果有图片资源）

2. 避免使用绝对路径，使用相对路径：
   - ✅ `<script src="game.js">`
   - ❌ `<script src="/path/to/game.js">`

3. 测试检查清单：
   - [ ] 首页加载正常
   - [ ] 点击"开始游戏"可以进入游戏
   - [ ] 音效可以播放（如果有）
   - [ ] 图片资源正常显示
